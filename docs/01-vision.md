# プロジェクトビジョン

## 何を作っているか

AI 同士が将棋で対戦し、人間がその思考プロセスを観戦・検証できるローカル Web アプリケーション。人間が参加してAIと対局することもできる。

## 差別化ポイント

**AI 時代ならではの思考プロセスの可視化**

単に強い AI を動かすのではなく、AI がどの手を候補にあげ、なぜその手を選び、局面をどう評価したかを棋譜ログとして残す。観戦者はリプレイを通じて AI の判断を追跡できる。

これは強い将棋 AI（stockfish 等）にはない視点であり、LLM 時代のアプリとして意味を持つ。

## 設計原則

- 合法手判定はサーバー側で必ず行う。AIがどのようなバックエンドであっても同じルールが適用される。
- 棋譜ログには内部推論ではなく、説明可能な要約（候補手・評価・理由）を保存する。
- AI の実装は差し替え可能（`AiPlayer` インターフェース）。SimpleAI・Ollama・外部エンジン・LLM を同じ仕組みで扱う。
- ブラウザ UI と HTTP API の両方から操作できる。Playwright MCP による外部エージェントも同じ経路を使う。

## AI 種別と特性

| 種別 | 仕組み | 速さ | 思考の透明性 |
|---|---|---|---|
| SimpleAI（深さ1） | ヒューリスティックスコア | 即座 | 高い（ルールが明快） |
| SimpleAI（深さ2〜4） | αβ法による手先読み | 速い〜数秒 | 高い（探索深さが明示される） |
| Ollama / LLM | 言語モデルへのプロンプト | 数秒〜十数秒 | 中（理由は自然言語で出力） |
| 外部エージェント | 任意（外部プロセス） | 任意 | 任意 |

SimpleAI は「昔の将棋ソフトのアルゴリズム」として教育的価値もあり、透明性の高い比較対象として残す。
